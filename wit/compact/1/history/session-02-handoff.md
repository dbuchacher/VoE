## What works

- Blue screen boots (VESA 1024×768×32)
- "nomos" renders in white at top-left (bitmap font via render_char)
- Speaker beep
- NVMe PCI discovery ('N' if found, 'n' if not)
- Keyboard scancodes to debugcon
- Headless fallback (skip filld when no VESA)
- BSS zeroed by bootloader (page tables in .pagetables section)

## What was built then reverted

A complex trace walk (keyboard → trace → trace read → output) was built and verified correct by serial execution trace, but caused QEMU TCG to produce spurious debugcon noise. The trace walk was reverted to the simple kbd_walk to fix a MBR boot crash. The trace primitive IS proven (round-trip test: write 42, read back '*'). The indirect flags (bit7/6 dereference) work. The infrastructure is ready — the walk just needs to be rebuilt carefully.

## What was built then deleted

- **ASCII scancode table** — violated the design ("Not ASCII"). Deleted. The proper path: scancode → wave byte directly. Wave byte computation should be lazy (compute on first use, cache in BSS table). Same pattern as get_atom.
- **wavemap.ha binary blob** — pre-baked wave byte table generated by Python. Wrong approach. Wave encoding should be computed at runtime by a walk, not pre-generated. Deleted along with gen-wavemap.py.
- **Claude memory files** — should never exist. All knowledge goes in wit/. Memory disabled in project settings.

## What's actually pending

1. **Scancode → wave byte** — the keyboard walk needs to convert PS/2 scancodes to wave bytes, not output raw scancodes. Needs the wave byte classification (T=anatomy, D=width, M=category, Q=case) computed lazily.
2. **Trace walk** — the kbd_walk should write wave bytes to the trace, read from trace, render to screen. Infrastructure exists (indirect flags, stash variables, trace registry). Was working but caused QEMU noise.
3. **Genesis splitting** — genesis still has the PCI probe inline (via %include). The BSS declarations for traces, stash, cursor, nvme_bar are all in genesis. Could move to a shared defs file.
4. **Bind drain** — needs multiple traces + parameterized walks.
5. **NVMe queue init** — BAR discovered, needs MMIO setup for submission/completion queues.

## Rules (READ THESE)

- **DON'T BLOAT FILES.** New walk → new file in 2-walks/. New table → new file. New scaffolding → new file. Genesis is ONE-TIME BOOT. Kernel is SEALED.
- **NO ASCII.** Characters are wave bytes. Not ASCII. Not Unicode. Read the design doc.
- **NO CLAUDE MEMORY FILES.** All knowledge lives in wit/. Auto memory is disabled.
- **Equation is sealed.** get_atom and emit_function never change. Walker can grow.
- **Read mind/coder/design** before coding. Read the "WHAT WE LEARNED" section.

## Key technical details

- Walker has indirect flags: bit7=deref arg0, bit6=deref arg1 in flags byte
- write(pipe, *stash) = flags 0x48. test(pipe, *stash) = flags 0x48
- read(addr) already dereferences — don't use indirect on read's arg0
- NASM labels compute skip/loop counts in walks — use .label_skip/.label_end patterns
- QEMU debugcon noise: trace walk produces spurious bytes on port 0xE9. Walk is correct. Use serial or GUI for testing.
- BSS zeroed by bootloader. Page tables in .pagetables section (not zeroed).
- Build outputs go to code/.build/ (gitignored)
